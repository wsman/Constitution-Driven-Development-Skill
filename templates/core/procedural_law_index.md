- **输入**: 条款编号 (如 "§201")
- **输出**: 对应的 WF-xxx 文件路径
- **算法**: 索引查找 → `workflows/WF-{section_number}_*.md`

### 2. 按关键词检索

### 3. 按类别检索

## CDD五状态工作流 (§201)

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ State A  │ ──→ │ State B  │ ──→ │ State C  │ ──→ │ State D  │ ──→ │ State E  │
│ 基准摄入 │     │ 文档规划 │     │ 受控执行 │     │ 三级验证 │     │ 收敛纠错 │
└──────────┘     └──────────┘     └──────────┘     └──────────┘     └──────────┘
```

### State A: 基准摄入 (Context Ingestion)
读取项目的"可执行规范"（单一真理源）

### State B: 文档驱动规划 (Documentation First)
**[法定动作 §152]**
- 严禁直接修改代码
- 先文档后代码
- 等待用户批准 (YES) 后继续

### State C: 受控执行 (Safe Implementation)
依据已批准的 Memory Bank 执行代码修改

### State D: 三级验证闭环 (Verification Loop)
**[验证即循环 §136]**
- Tier 1: 结构验证
- Tier 2: 签名验证
- Tier 3: 行为验证

### State E: 收敛与纠错 (Convergence)
- 任何验证失败 → 修正代码或文档
- 全部通过 → 完成任务，记录成果
